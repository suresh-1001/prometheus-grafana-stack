## 4. Alerting

- [ ] `config/alertmanager.yml` — all `!! CONFIGURE !!` placeholders replaced
- [ ] SMTP credentials tested (`amtool check-config /etc/alertmanager/alertmanager.yml`)
- [ ] Slack webhook URL verified and tested
- [ ] Alert rules visible at `http://<host>:9090/alerts`
- [ ] Test alert fired and received in email and Slack

---

## 5. Security

- [ ] Grafana default admin password changed from `changeme`
- [ ] `.env` file is NOT committed to Git (confirmed in `.gitignore`)
- [ ] Prometheus and Alertmanager ports restricted to internal network / VPN only
  (Grafana on 3000 is the only port that should face the internet if at all)
- [ ] Firewall rules verified — only required ports open

---

## 6. Data Retention & Storage

- [ ] Prometheus retention confirmed at 30 days (`--storage.tsdb.retention.time=30d`)
- [ ] Sufficient disk space for TSDB growth (`df -h /var/lib/prometheus` or `docker volume inspect`)
- [ ] Named volumes in use (Docker) — no anonymous volume data loss risk

---

## 7. Bare Metal Only

- [ ] All four systemd services enabled and running
  ```
  systemctl status prometheus alertmanager node_exporter grafana-server
  ```
- [ ] Services survive reboot — tested with `systemctl reboot` or confirmed `WantedBy=multi-user.target`
- [ ] Full install log reviewed at `/var/log/monitoring-install.log`
